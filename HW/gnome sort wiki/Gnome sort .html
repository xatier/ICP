[编辑] 地精排序

地精排序, 由 Hamid Sarbazi-Azad 在 2000 年提出 Stupid sort, 之後被 Dick Grune 叫作 "Gnome sort"[1] 這是一個 排序演算法 sorting algorithm 和插入排序 insertion sort 類似. 這是一個概念簡單，沒有很潮的巢狀迴圈的演算法. 其時間複雜度為(n²)

[编辑] 步驟

[编辑] 地精排序是起源於花園裡可愛的地精們排序一列花盆的神奇外星科技

想像陣列 a[n] 裡的元素是我們的花盆，地精排序是依照下列步驟進行

1) 花園地精會依序便歷所有花盆，從 a[0] 到 a[n-1]，挑起兩個相鄰的花盆 a[i] 和 a[i+1]
2) 如果是正確的順序，那就往下一組花盆看看
3) 假若是錯誤的順序，那就交換一下吧！並往前一組花盆看看
4) 如果已經沒有花盆了，那就直接往前找下一組
5) done!


[编辑] 虛擬碼

procedure gnomeSort(a[])
    pos := 1
    while pos < length(a)
        if (a[pos] >= a[pos-1])
            pos := pos + 1
        else
            swap a[pos] and a[pos-1]
            if (pos > 1)
                pos := pos - 1
            else
                pos := pos + 1
            end if
        end if
    end while
end procedure


[编辑] C 語言實作

人、人家才不會承認這是小黃的作業

 void gnomesort(int n) {
   int i;
   i = 1;
   while (i < n) {
     if (a[i] >= a[i-1]) {              // we are in order! yeah!
       i++;
     }
     else if (a[i] < a[i-1]) {          // ohoh, we need swap
       int tmp = a[i];
       a[i] = a[i-1];
       a[i-1] = tmp;
       i--;
     }
     else if (i <= 1) {                // go ahead!
       i++;
     }
     else{}
   }
 }


[编辑] 複雜度

神奇地精排序平均所花費的時間複雜度是都是O(n2)，但如果序列在一開始已經大部分排序過的話，會接近O(n)。
